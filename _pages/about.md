Welcome to my Computer Science capstone ePortfolio.

I'm Brian Zavala, a Computer Science student at Southern New Hampshire University completing the CS 499 Capstone. This site is the culmination of my undergraduate program — a curated showcase of my growth as a software engineer, presented as a single artifact enhanced across the three core areas of computer science.

## The Capstone Artifact

My chosen artifact is **Travlr Getaways**, a full-stack travel booking platform originally built in CS 465 (Full-Stack Development I). Travlr is a MEAN-stack application: a Node/Express backend serving both a server-rendered public site and a secure Angular single-page admin interface, backed by MongoDB and secured with JWT authentication.

I selected Travlr because a full-stack application naturally demonstrates expertise across all three capstone categories. Rather than spread myself across three separate projects, I'm enhancing one cohesive system to show depth as well as breadth.

## Three Enhancement Categories

- **Software Design and Engineering** — Unifying the frontend architecture, adding proper error handling, environment-based configuration, and a test suite.
- **Algorithms and Data Structures** — Implementing server-side search, filtering, sorting, cursor-based pagination, and an in-memory cache with documented Big-O trade-offs.
- **Databases** — Adding indexes and schema validation, building aggregation pipelines, hardening authentication, and introducing an audit log to demonstrate a security mindset.

Each enhancement is documented in its own project page, paired with a narrative explaining the motivation, the skills demonstrated, and the course outcomes it addresses.

## What You'll Find Here

- The original and enhanced Travlr artifact across all three categories
- A recorded code review walking through the existing code and planned improvements
- Narratives reflecting on each enhancement's process, challenges, and learning
- A professional self-assessment introducing my skills and career direction

This site is maintained throughout the course and will be complete at the end of Module 7.
